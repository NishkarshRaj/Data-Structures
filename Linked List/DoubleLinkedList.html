Theory

 
A Doubly Linked List is a Linked List which has connection pointer to each of its previous element and is generally defined as "prev"

Thus, a node in Doubly Linked Pointer consists of following:

	1) next pointer => contains the address of next node of the DLL.
	2) data elements => contains the information of the node of DLL.
	3) prev pointer => contains the address of the previous node of the DLL.

Declaration:
class classname
{
datatype data1,....; //data elements
classname *next; //next pointer
classname *prev; //previous pointer
}

Complexity
  1. Insertion
    • Worst case time complexity O(n)
    • Best case time complexity O(1)
    • Average case time complexity O(1)
    • Space complexity O(1)
  2. Deletion
    • Worst case time complexity O(n)
    • Best case time complexity O(1)
    • Average case time complexity O(1)
    • Space complexity O(1)
  3. Display
    • Worst case time complexity O(n)
    • Best case time complexity O(n)
    • Average case time complexity O(n)
    • Space complexity O(1)

Applications of Doubly Linked List
  1. Redo-Undo options
  2. Navigating next/previous pages on Web
  3. Sorting history by most recent.

Advantages of Doubly Linked List
  1. A Doubly linked list can be traversed from both its head and its tail.
  2. Deletion Operations becomes efficient and uses less memory.
  3. For a sorted Doubly Linked list, it is more efficient to traverse from nearest end.

Disadvantages of Doubly Linked List
  1. It uses one extra pointer for all the nodes, thus increasing the space complexity and security concerns.
  2. Handling large number of pointers while performing deletion is very tedious.
